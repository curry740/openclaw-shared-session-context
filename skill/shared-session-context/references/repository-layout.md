# Repository Layout

## Recommended layout if published separately

- `SKILL.md`
- `scripts/bootstrap-config.sh`
- `scripts/check-env.sh`
- `references/integration-checklist.md`
- `references/publish-plan.md`
- `references/repository-layout.md`

Keep it small.
Do not add README, changelog, or user-facing install docs inside the skill package.
Those belong in the standalone repository.
